2014-04-13, Version 0.9.0 Released

Most prominent changes include:

  • flow: Added support for builtin handler/function overloading.
  • flow: Added support for array assignments to variables.
  • flow: Added support for unary bit-wise NOT operator.
  • flow: Improved internal constant value handling.
  • flow: Improved array handling, also resulting into runtime performance boosts and better semantic verifications.
  • flow: Improved interpreter loop performance (uses direct threading).
  • flow: Improved IR-optimization passes & enabled optimizations by default
  • flow: Improved builtin callee user-verification to use (optimized) IR instead of AST now.
  • flow: Dropped superfluous VM opcodes.
  • flow: Fixes to all boolean operators.
  • flow: Fixes use of large integer constants in expressions.
  • flow: Fixes "superfluous params" error message.
  • x0d: builtin workers() is now overloaded for workers(I)V and workers(i)V; no workers.affinity(i)V anymore.
  • x0d: memory resource leak fixes upon program exit.
  • x0d: Fixed precompressed() handler.
  • plugins: new plugin "access" for basic IP & CIDR network based access control.
  • director: Adds an option to directly respond with 503 instead of enqueuing the request when the complete cluster is down.
  • director: Fixed bug in HTTP and FastCGI backends that caused hanging client connections on offline backends.
  • accesslog: Uses non-blocking log-file writes. Improves performance within the networking code.
  • doc: x0d man-page extended to describe error severities.
For a complete list of changes, please check https://github.com/xzero/x0/commits/

2014-03-23, Version 0.8.0 Released

Most prominent changes include:

  • flow: Flow Configuration parser and backend rewritten
  • flow: All general statements must end with a ; at the end.
  • flow: Buildin handler and functions as well as local variables are now strongly typed.
  • flow: Adds Cidr notations as literal values for IPv4/IPv6 networks, such as 192.168.0.0/16
  • flow: Adds nested configuration file parsing, ie. #include "path"
  • flow: Adds new keyword match for optimized multi-branch pattern matching
  • flow: Native handler/function calling syntax improved to support defaulted and mixed arguments.
  • flow: Dropped dependency on LLVM and use a special-purpose virtual machine for running the Flow programs.
  • http: Various bug fixes, such as HTTP request buffer management.
  • director: Added experimental response object caching support.
  • director: Added support for reconfiguring scheduling algorithms (also at runtime).
  • director: Fixes 2 bugs related to HTTP/FastCGI backend connection reference counting.
  • x0d: "vhost"-plugin removed as the new Flow match-statement syntax is superior.
  • x0d: Adds request_header_buffer_size config setup variable.
  • x0d: Adds request_body_buffer_size config setup variable.
  • x0d: CLI parameters extended and improved. Check x0d --help for details.
  • x0d: Adds severity diag for logging diagnostics, generally helpful for web developers and operators.
  • x0d: Also added builtin Flow function log.diag(message).
  • doc: manual pages generally improved
For a complete list of changes, please check https://github.com/xzero/x0/commits/
